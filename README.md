# copa

> Generate clean C header files directly from C source files.

`copa` is a lightweight C header generator that parses `.c` source files and automatically produces matching `.h` files with include guards.

Instead of manually maintaining header files, `copa` extracts public declarations—including functions, structs, typedefs, macros, includes, function pointers, and extern variables—and generates a clean, deduplicated header.

**Version:** 0.1.1

---

# Example

Given the following source file:

```c
// foo.c

#include <stdio.h>

#define MAX_SIZE 1024
#define SQUARE(x) ((x) * (x))

typedef struct {
    int x;
    int y;
} Point;

typedef void (*callback_t)(int, const char *);

extern int global_var;

int add(int a, int b);

int *get_pointer(const char *name)
{
    ...
}
```

Running

```bash
copa foo.c
```

produces

```c
/* Generated by copa
 * https://github.com/ka1rav6/copa
 */

#ifndef FOO_H
#define FOO_H

#include <stdio.h>

#define MAX_SIZE 1024
#define SQUARE(x) ((x) * (x))

typedef struct {
    int x;
    int y;
} Point;

typedef void (*callback_t)(int, const char *);

extern int global_var;

int add(int a, int b);

int *get_pointer(const char *name);

#endif /* FOO_H */
```

`copa` automatically:

- strips function bodies
- preserves public declarations
- removes duplicate prototypes
- preserves typedefs
- preserves macros
- combines anonymous `typedef struct { ... } Name;` declarations correctly

# How it Works

`copa` uses a four-stage compilation pipeline.

```
Source (.c)
      │
      ▼
 Lexer
      │
      ▼
 Flattener
      │
      ▼
 Parser
      │
      ▼
 Generator
      │
      ▼
Header (.h)
```

## 1. Lexer

Converts source text into tokens.

Recognizes

- identifiers
- keywords
- literals
- operators
- preprocessor directives
- comments
- punctuation

---

## 2. Flattener

Transforms the lexer output into a single token stream by

- removing comments
- removing newlines
- merging token vectors

This greatly simplifies parsing.

---

## 3. Parser

Builds an abstract syntax tree (AST) of top-level declarations.

Supported declarations include

- functions
- structs
- unions
- enums
- typedefs
- function pointers
- macros
- includes
- extern variables
- forward declarations

---

## 4. Generator

Walks the AST and emits a clean header file.

The generator automatically

- inserts include guards
- strips function bodies
- removes duplicate prototypes
- preserves declaration order
- formats output consistently

# Usage

```
copa [-r] [--watch] [--pragma-once] [--indent <2|4|8>] <file.c | directory>
```

## Commands

| Command | Description |
|----------|-------------|
| `copa foo.c` | Generate `foo.h` |
| `copa src/` | Generate headers for every `.c` file |
| `copa -r src/` | Generate headers recursively |
| `copa --watch src/` | Watch directory for changes |
| `copa -r --watch src/` | Recursive watch mode |
| `copa --pragma-once foo.c` | Use `#pragma once` instead of include guards |
| `copa --indent 2 src/` | Use 2-space indentation |

## Formatting Options

| Flag | Description | Default |
|------|-------------|---------|
| `--pragma-once` | Use `#pragma once` instead of `#ifndef`/`#define` guards | Include guards |
| `--indent <N>` | Indent width (2, 4, or 8) | 4 |

Generated headers are written beside their corresponding source files.

Existing headers are overwritten.

Non-C files are ignored automatically.

# Supported C Constructs

## Includes

```c
#include <stdio.h>
#include "foo.h"
```

---

## Macros

```c
#define SIZE 100

#define MAX(a,b) ((a) > (b) ? (a) : (b))
```

---

## Structs

```c
struct Node {
    int value;
    struct Node *next;
};
```

Anonymous typedefs

```c
typedef struct {
    int x;
    int y;
} Point;
```

---

## Unions

```c
union Value {
    int i;
    float f;
};
```

---

## Enums

```c
enum Color {
    RED,
    GREEN,
    BLUE
};
```

---

## Typedefs

typedef unsigned long ulong;

# Watch Mode

Watch mode scans the target directory every **500 ms**.

Whenever a `.c` file changes,

- it is reparsed
- its header is regenerated

Recursive watching is available with `-r`.
